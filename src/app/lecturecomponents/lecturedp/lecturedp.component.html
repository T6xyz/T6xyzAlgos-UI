Intro to Dynamic Programming

Dynamic Programming is a topic that is often considered one of the most difficult concept to master in algorithms. It is not uncommon to hear someone discuss how they encountered an extremely hard dynamic programming problem in an interview or competitive coding contest. Dynamic Programming is often seen as a daunting concept to master and many lecturers disucss Dynammic Programming in an abstract manner, leaving many students confused. While Dynamic Programming can be difficult to grasp, these next few lectures will discuss a variety of different Dynamic Programming topics in-depth to help simplify the concepts.

What is Dynamic Programming?

Dynamic Programming is an optimization technique used to solve complex problems by dividing them into smaller, simpler sub-problems and using the solutions to those sub-problems to solve a larger, more complex problem. Often times, many brute-force algorithms have an exponential or factorial runtime, which for large sized inputs, can lead to slow computations that might take a long time to run. Problems that can be solved using Dynamic Programming contain the following properties:


1. The problem can be divided into smaller, simpler sub-problems. Those sub-problems can then be used to find the solution to a larger, more complex problem.


2. The problem has overlapping sub-problems. Rather than recomputing solutions to sub-problems that have already been solved, dynamic programming utilizes a data structure to cache the solutions to sub-problems that have already been computed, removing redundant computations and decreasing overall run-time.


Knowing these key ideas, we can leverage these two properties to create an efficient Dynamic Programming solution. Now you might be wondering, how do we create a Dynamic Programming solution? What makes up a Dynamic Programming solution? We will discuss this in the next section below.


*For the rest of the lecture on Dynamic Programming, we will refer to Dynamic Programming as "DP" for simplicity.
